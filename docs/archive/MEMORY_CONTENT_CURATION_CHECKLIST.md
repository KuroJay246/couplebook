# Memory Content Curation Checklist

Date: 2026-07-08
Scope: review checklist only. No memory deletion or dataset rewrite in this phase.

## Goal

Create a safe editorial checklist for improving Couple Book memory quality later without deleting current data prematurely.

## Identify Technical / Raw Entries

Review entries that look mostly import-driven:

- timestamp-only titles like `Photo from ...`
- timestamp-only titles like `Video Clip ...`
- generic descriptions with no human context
- repeated media paths producing multiple near-duplicate entries
- source-heavy tags with little story value

## Identify Real Milestone Entries

Mark entries that already represent real product-worthy moments:

- anniversaries
- birthdays
- confession
- Valentine
- first-date or first-home style moments
- clearly meaningful couple events

## Identify Special-Page Entries

Keep a separate list for:

- confession
- Valentine
- birthday

Later decisions should define whether they remain:

- timeline entries
- home feature entries
- special-moments hub entries

## Identify Missing Captions

Flag entries that need:

- a real title
- a better caption
- a short story line
- a mood/category label

## Identify Local-Only Media Dependencies

Mark entries that depend on:

- local-only photo paths
- local-only video paths
- excluded special-page companion media

These should be planned carefully before any Storage work.

## Decide Which Memories Should Be Featured On Home

Later editorial pass should select:

- featured milestone memories
- warm everyday memories worth resurfacing
- special moments worth seasonal resurfacing
- gallery highlights that deserve story framing

## No Data Deletion Without Approval

Until a later reviewed migration phase:

- do not delete memories for being messy
- do not remove imports just because they are technical
- classify first, then preview cleanup choices
